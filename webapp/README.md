# 🚦 Светофор — веб-сервис

Автономный веб-дэшборд статуса delivery по стримам и командам. Показывает
светофор (🟢🟡🔴) по метрикам, распределение времени по стадиям (где узкое место),
сигналы проблем и рекомендации командам. Всё считается **детерминированным
Python-кодом** (без ИИ), из Jira.

Открыть (когда запущен): **http://localhost:8787**

---

## 🟢 Для нетехнической команды

### Как добавить/убрать стрим или переименовать
Файл **`config/dashboard.json`**, секция `streams` — только для НАЗВАНИЙ (сами
стримы подтягиваются автоматически: все проекты Jira с эпиками). Тут можно задать
понятное имя вместо технического:
```json
"streams": [
  {"project": "AIHUB",    "label": "AI HUB"},
  {"project": "CMASSORT", "label": "Ассортимент"}
]
```
Если стрима нет в списке — он всё равно покажется, но с названием проекта из Jira.

### Как поменять пороги цветов
Файл **`config/zones.json`** — см. **[config/README.md](../config/README.md)**.
Меняешь число (например, LT зелёный до 14 дней) → светофор перекрашивается. Кода не трогаешь.

### Как это читать
- **Слева** — список всех стримов (поиск, цветной кружок = статус, число = просрочки).
- **Клик по стриму** → метрики, «где теряется время» (стадии), сигналы, команды.
- **Клик по команде** → спринт-прогресс, что под риском, рекомендации что делать.
- Серый кружок = детализация ещё считается в фоне (появится через минуту).

## 🔧 Для технической команды

### Запуск
```bash
python3 webapp/server.py       # → http://localhost:8787
```
Нужны: VPN (Jira) и заполненный `.env` (сервис сам его подхватывает).

Переменные окружения (необязательные):
| Переменная | По умолчанию | Что задаёт |
|---|---|---|
| `DEVPULSE_PORT` | 8787 | порт |
| `DEVPULSE_LIGHT_MIN` | 20 | как часто пересобирать лёгкий список стримов (мин) |

### Архитектура: почему сайт быстрый
Живой сервер, который считал бы всё на каждый запрос, был бы медленным (Jira —
узкое место). Поэтому **фоновый расчёт + кэш**:

```
┌─ light-воркер (каждые ~20 мин) ── список ВСЕХ стримов + дешёвые счётчики (быстро)
├─ heavy-воркер (непрерывно) ────── обходит стримы, считает полную детализацию
│                                    (LT/TTM, стадии, сигналы, команды) → кэш + снимок на диск
└─ веб-сервер ───────────────────── отдаёт из кэша МГНОВЕННО (SPA + JSON API)
```
- Пользователь ничего не ждёт: данные посчитаны заранее.
- Снимки на диске (`webapp/cache/`) переживают перезапуск.
- Полный цикл детализации по всем стримам ~1 час; стримы идут «сначала проблемные»
  (по просрочкам) и кэшируются по мере расчёта. Ещё не посчитанные показывают
  базовые KPI + «считается».

### Команда = доска Jira (100% точно)
Команда определяется **не** парсингом текста имени спринта (у каждого стрима свой
формат), а по **доске**: у спринта есть `rapidViewId`, у доски — имя (Agile API
`/rest/agile/1.0/board/{id}`). Это реальный объект Jira. См. `devpulse/sprints.py`.

### API
| Endpoint | Что отдаёт |
|---|---|
| `GET /api/streams` | лёгкий список всех стримов (для боковой панели) |
| `GET /api/stream/{PROJECT}` | полная детализация стрима (или `{computing:true}`, если ещё считается) |
| `GET /api/team/{PROJECT}/{TEAM}` | детализация команды (спринт + рекомендации) |
| `GET /` , `/static/*` | сам сайт (SPA) |

### Файлы
| Файл | Что делает |
|---|---|
| `server.py` | Веб-сервер (stdlib), два фоновых воркера, кэш, снимки на диск |
| `model.py` | Сборка данных: `discover_streams()` (лёгкий), `build_stream_full()` (полный) |
| `static/index.html` | Каркас страницы |
| `static/style.css` | Стили (тема свет/тьма) |
| `static/app.js` | SPA: боковая панель, роутинг обзор→стрим→команда, рендер |
| `cache/` | Снимки данных (в .gitignore) |

Вся математика — в пакете `devpulse/` (переиспользуется), сам `webapp/` только
собирает и показывает.

### Альтернатива: статический файл
`python3 build_dashboard.py` (в корне) печёт самодостаточный `dashboard.html`
(данные вшиты). Без сервера, но снимок — нужно пересобирать. Полезно как отчёт/leave-behind.

## Прод-вынос (на будущее)
Сейчас сервер на ноуте (localhost) — работает, пока машина включена и VPN поднят.
Для настоящей автономности (постоянный URL): вынести `webapp/` на **ВМ** и запустить
как сервис (systemd/launchd) под сервисным аккаунтом вместо личного токена. Код к
этому готов — переносится как есть.

### Оптимизация скорости
Полный цикл ~1 час упирается в последовательный обход стримов. Ускорение —
распараллелить heavy-воркер (пул из 3–4 потоков) → цикл ~15–20 мин.
