# Настройка светофора — для всей команды (без программирования)

Здесь лежат **настройки**, которые можно менять без кода. Правите числа и текст
в `.json`-файлах, коммитите в git — светофор подхватит. Логику (как считается)
трогать не нужно, она в `devpulse/svetofor/`.

> Правило: меняйте **значения** (числа, true/false, текст в note). НЕ меняйте
> названия полей и структуру скобок `{ }`. После правки — проверьте, что файл
> не «сломан»: онлайн-валидатор JSON или спросите технического.

---

## `zones.json` — пороги зон 🟢🟡🔴

Для каждой метрики задаются границы зелёной/жёлтой/красной зоны.

**Метрика «чем меньше, тем лучше»** (`lower_better`, напр. Lead Time, TTM):
```
"green_max": 14,   // ≤ 14 → 🟢 зелёный
"yellow_max": 21   // ≤ 21 → 🟡 жёлтый, больше → 🔴 красный
```

**Метрика «чем больше, тем лучше»** (`higher_better`, напр. Предсказуемость %):
```
"green_min": 80,   // ≥ 80 → 🟢
"yellow_min": 60   // ≥ 60 → 🟡, меньше → 🔴
```

**Что можно менять безопасно:**
- числа порогов (`green_max`, `yellow_max`, `green_min`, `yellow_min`);
- `"enabled": true/false` — включить/выключить метрику в светофоре;
- `"note"` — комментарий (для себя и коллег);
- `"interpretation"` у predictability — какую трактовку формулы берём (см. ниже).

**Кастом по подразделению** — в `overrides_by_scope`. Пример: у AIHUB своя цель по LT:
```
"overrides_by_scope": {
  "AIHUB": { "lead_time": { "green_max": 10 } }
}
```

**Заглушки:** метрика с `"enabled": false` (напр. Чистота данных) в светофоре
показывается серым и на цвет не влияет — пока нет расчёта. Как появится — ставим
`true`.

## `dashboard.json` — какие стримы показывать на сайте

Стримы (проекты Jira) подтягиваются автоматически — все с эпиками. В `dashboard.json`
задаются только **понятные названия** вместо технических ключей:
```
"streams": [
  {"project": "AIHUB",    "label": "AI HUB"},
  {"project": "CMASSORT", "label": "Ассортимент"}
]
```
Стрим без записи всё равно покажется — с названием проекта из Jira.

## `aggregation.json` — как свести метрики в один цвет

```
"rule": "worst"   // цвет светофора = по худшей метрике (рекомендуется)
```
- `worst` — если хоть одна метрика красная, весь светофор красный. Консервативно, честно.
- `grey_policy`: `ignore` — метрики без данных не влияют; `strict` — любая серая делает светофор серым.

## Текущие значения — что провизорно, а что подтверждено

| Метрика | Порог | Статус |
|---|---|---|
| Lead Time | 🟢≤14 🟡≤21 | из Базы знаний, НЕ подтверждено Димой |
| TTM | 🟢≤57 🟡≤75 | провизорно (среднее ~49-57 дн) |
| Predictability | 🟢≥80 🟡≥60 | **сейчас выключена** (`enabled:false`): формула не зафиксирована |
| Чистота данных | 🟢≥90 🟡≥70 | **выключена** (`enabled:false`): расчёт не построен |

Всё это — **черновик, чтобы стартовать**. По мере подтверждения методологии
меняете числа здесь, без правки кода.
